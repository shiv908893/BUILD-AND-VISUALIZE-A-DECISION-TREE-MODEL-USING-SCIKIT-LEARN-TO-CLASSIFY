*DESCRIPTION*:

TO BUILD AND VISUALIZE A DECISION TREE FOR CLASSIFYING CAR ORIGINS (USA VS. NON-USA) BASED ON THE CAR_SALES DATASET, WE FOLLOW A STANDARD MACHINE LEARNING PIPELINE: DATA PREPROCESSING, MODEL TRAINING, AND EVALUATION.
DECISION TREE IMPLEMENTATION
THE FOLLOWING PYTHON CODE USES SCIKIT-LEARN TO PROCESS THE DATA, TRAIN A DECISIONTREECLASSIFIER, AND VISUALIZE THE RESULTING LOGIC. WE SELECTED 14 NUMERIC FEATURES (SUCH AS PRICE, HORSEPOWER, AND WEIGHT) TO PREDICT WHETHER A CAR IS FROM THE USA OR A NON-USA MANUFACTURER.
MODEL VISUALIZATION AND ANALYSIS
1. VISUAL INTERPRETATION
THE DECISION TREE VISUALIZES THE "IF-THEN" LOGIC THE MODEL USES TO CLASSIFY CARS.
ROOT NODE: THE MOST IMPORTANT FEATURE USED FOR THE FIRST SPLIT IS USUALLY A PHYSICAL DIMENSION OR PERFORMANCE METRIC (E.G., LENGTH OR WEIGHT).
COLOR CODING: IN THE VISUALIZATION, NODES ARE COLORED BASED ON THEIR MAJORITY CLASS. A DEEPER COLOR INDICATES HIGHER "PURITY" (CERTAINTY) IN THAT SPECIFIC LEAF.
DECISION RULES: FOR EXAMPLE, A RULE MIGHT STATE: IF A CAR'S LENGTH IS LESS THAN 188.5 INCHES AND ITS PRICE IS BELOW $15,000, IT IS MORE LIKELY TO BE A NON-USA CAR.
2. PERFORMANCE METRICS
ACCURACY: THE MODEL ACHIEVED APPROXIMATELY 61% ACCURACY ON THE TEST SET. GIVEN THE SMALL SIZE OF THE DATASET (93 RECORDS), THIS IS A REASONABLE BASELINE.
PRECISION/RECALL: THE MODEL PERFORMED SLIGHTLY BETTER AT IDENTIFYING USA CARS THAN NON-USA CARS, THOUGH THE SMALL SAMPLE SIZE MEANS PERFORMANCE CAN FLUCTUATE BASED ON THE DATA SPLIT.
3. STRATEGIC INSIGHTS
DECISION TREES ARE HIGHLY INTERPRETABLE. BY LOOKING AT THE TREE, MANUFACTURERS OR ANALYSTS CAN SEE WHICH SPECIFIC FEATURES DIFFERENTIATE DOMESTIC CARS FROM IMPORTS. IN THIS DATASET, DIMENSIONS LIKE LENGTH AND WHEELBASE OFTEN SERVE AS STRONG DISCRIMINATORS, REFLECTING DIFFERENT DESIGN STANDARDS BETWEEN REGIONS.
